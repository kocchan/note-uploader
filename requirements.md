# Phase 1: MVP要件定義書

## 概要

ADHD向けのAI活用術をテーマにしたnote有料記事ビジネスを、AIエージェントで自動化するシステムのPhase 1（MVP）。

**記事価格: 1,980円**（この価格に見合う質のコンテンツを作成する）

---

## システム構成

```
┌─────────────────────────────────────────────────────┐
│           オーケストレーター（Claude Code）           │
│              全体の進行管理・エージェント連携           │
└─────────────────────────────────────────────────────┘
                         │
    ┌────────────────────┼────────────────────┐
    ↓                    ↓                    ↓
┌─────────────┐  ┌─────────────────┐  ┌─────────────┐
│    戦略     │  │  コンテンツ作成   │  │    X運用    │
│（3エージェント）│ │ （3エージェント）  │  │（2エージェント）│
└─────────────┘  └─────────────────┘  └─────────────┘
    │                    │                    │
    ↓                    ↓                    ↓
┌─────────────────────────────────────────────────────┐
│              出力: Markdownファイル                  │
└─────────────────────────────────────────────────────┘
```

## 戦略系エージェント

### 1. トレンド収集エージェント

**目的**
取り入れられそうなトレンドを収集・整理

**入力**
- note APIからのトレンド情報
- Web検索結果（ADHD × AI関連）
- 人間が提供する参考URL

**処理内容**
1. noteのトレンド記事を調査
2. ADHD × AI活用の最新トピックを収集
3. 取り入れられそうなネタを抽出
4. トレンドの鮮度・需要を評価

**出力**
```
output/strategy/
  YYYYMMDD_trend_report.md    # トレンドレポート
```

**出力フォーマット例: trend_report.md**
```markdown
# トレンドレポート - YYYY/MM/DD

## 今週のトレンド

### 1. Claude 3.5 Sonnetの活用法
- 需要: 高
- 鮮度: 今週話題
- ADHD向け切り口: タスク分解の精度が向上

### 2. AI × 習慣化
- 需要: 中
- 鮮度: 継続的に人気
- ADHD向け切り口: 三日坊主をAIで防ぐ

### 取り入れ推奨ネタ
- ...
```

### 2. 記事分析エージェント

**目的**
いいねが多い記事・購入されている記事の構成と書きっぷりを分析

**入力**
- 人気記事のURL（人間が提供 or API取得）
- 購入数/いいね数が多い記事リスト

**処理内容**
1. 記事の構成（目次・見出し）を抽出
2. 書きっぷり（文体・トーン）を分析
3. 無料/有料の切り分けポイントを分析
4. 価格設定と内容のバランスを評価
5. 成功パターンを抽出

**分析観点**
| 観点 | チェックポイント |
|------|----------------|
| 構成 | 見出し数、無料/有料の比率、導入の長さ |
| 書きっぷり | 口調、共感表現、具体例の量 |
| 価値提供 | 有料部分の具体度、コピペで使えるか |
| 導線 | CTAの配置、購入への誘導方法 |

**出力**
```
output/strategy/
  YYYYMMDD_article_analysis.md    # 記事分析レポート
```

**出力フォーマット例: article_analysis.md**
```markdown
# 記事分析レポート

## 分析対象
- 記事A: 「ADHDのタスク管理術」（いいね: 500、推定購入: 100）
- 記事B: ...

## 成功パターン

### 構成
- 無料部分: 全体の30%程度
- 導入: 共感から入る（「あなたもこうじゃないですか？」）
- 有料部分: すぐ使えるテンプレート/プロンプト

### 書きっぷり
- 一人称: 「俺」「私」（親しみやすさ）
- 語尾: 「〜なんですよね」「〜だった」混在
- 段落: 短め（2-3文）

### 価格と価値
- 1,980円帯: プロンプト5個以上 + 使い方解説
- 購入者の声: 「すぐ使えた」「具体的」

## 自記事への適用ポイント
- ...
```

### 3. 内省エージェント

**目的**
出版した記事の反応を分析し、PDCAを回す

**入力**
- 出版した記事の情報
- 記事作成時の「狙い」（戦略ドキュメント）
- 反応データ（いいね数、購入数、コメント）
- X投稿への反応

**処理内容**
1. 狙いと結果の比較
2. 良かった点・改善点の抽出
3. 次回への学びをまとめる
4. 成功/失敗パターンの蓄積

**出力**
```
output/strategy/
  YYYYMMDD_reflection.md    # 内省レポート
```

**出力フォーマット例: reflection.md**
```markdown
# 内省レポート - 記事「先延ばし癖をChatGPTで潰す方法」

## 記事情報
- 公開日: YYYY/MM/DD
- 価格: 1,980円
- URL: https://note.com/xxx

## 狙い（作成時の戦略）
- ターゲット: 先延ばしに悩むADHD当事者
- 差別化: 具体的なプロンプト5選
- 期待: 初週で10部販売

## 結果
- いいね: 50
- 購入数: 8部
- コメント: 3件（ポジティブ）
- X反応: RT 20、いいね 100

## 分析

### 良かった点
- 導入の共感が刺さった
- プロンプトが具体的で評価された

### 改善点
- タイトルがやや弱かった
- 無料部分が長すぎて有料への誘導が弱い

## 次回への学び
- タイトルに数字を入れる
- 無料部分は25%以内に
- 「すぐ使える」を強調

## 成功パターン蓄積
- 共感導入 → 効果あり
- プロンプト形式 → 購入につながる
```

## コンテンツ作成系エージェント

### 4. 構成作成エージェント

**目的**
承認されたテーマから記事のタイトルと目次を作成

**入力**
- 人間が承認したテーマ
- テーマの詳細情報（ターゲット、差別化ポイントなど）
- **狙いドキュメント**（なぜこの記事を書くのか）

**処理内容**
1. 複数のタイトル案を生成
2. 記事の目次（見出し構成）を作成
3. 無料/有料の境界線を提案
4. 各セクションの概要を記述

**出力**
```
output/articles/YYYYMMDD_テーマ名/
  intent.md     # 狙いドキュメント（内省用）
  outline.md    # タイトル案 + 目次
```

**出力フォーマット例: intent.md**
```markdown
# 記事の狙い

## テーマ
先延ばし癖をChatGPTで強制的に潰す方法

## なぜこの記事を書くのか
- トレンド: ChatGPT活用が伸びている
- 需要: 「先延ばし ADHD」の検索が多い
- 差別化: 具体的プロンプトを提供する記事が少ない

## ターゲット
- ADHDで先延ばし癖に悩む社会人
- AIツールは使ったことあるが活用できていない人

## 期待する成果
- 初週: 10部販売
- いいね: 100以上
- X拡散: RT 30以上

## 成功の定義
- 購入者から「使えた」のコメント
- リピート購入につながる

## 反省時のチェックポイント
- タイトルのクリック率
- 無料→有料の転換率
- 購入後の満足度（コメント）
```

**出力フォーマット例: outline.md**
```markdown

## テーマ
先延ばし癖をChatGPTで強制的に潰す方法

**入力**

- 人間からのレビューコメント・修正指示
- 参考情報・資料

**処理内容**
1. 各セクションの本文を執筆
2. プロンプトテンプレートの作成
3. 具体例・実行例の作成
4. 1,980円の価値を意識した内容量

**出力**
```
output/articles/YYYYMMDD_テーマ名/
  draft.md      # 記事本文（無料+有料）
  prompts.md    # プロンプトテンプレート集

### 6. 記事レビューエージェント

**目的**
記事のAIっぽさをチェックし、人間らしい文章に修正

**入力**
- 本文作成エージェントが出力した記事

**処理内容**
1. AIっぽい表現のチェック
   - 過度に丁寧な表現
   - 不自然な言い回し
   - テンプレート的な構成
   - 感情のない説明調
2. 人間らしさの追加
   - 口語的な表現
   - 具体的な体験談風の記述
   - 感情・共感の表現
3. 読者目線でのチェック
   - ADHD当事者として共感できるか
   - 1,980円の価値があるか

**チェック観点**
| 観点 | NGパターン | 修正例 |
|------|-----------|--------|
| 書き出し | 「〇〇について解説します」 | 「正直、これで人生変わった」 |
| 接続詞 | 「また」「さらに」の連続 | 段落を短く、リズムをつける |
| 語尾 | 「〜です」「〜ます」の単調さ | 「〜なんですよね」「〜だった」を混ぜる |
| 説明 | 網羅的・教科書的 | 「ぶっちゃけ、これだけでOK」 |

**出力**
```
output/articles/YYYYMMDD_テーマ名/
  review_report.md   # レビュー結果・修正点リスト
  draft_revised.md   # 修正済み記事
```

**次のステップ**
→ 人間が最終確認 → noteに手動投稿

## X運用系エージェント

### 7. X拡散エージェント

**目的**
ADHD関連アカウントに引用RTしてもらうためのツイートを作成

**入力**
- ターゲットアカウントの情報（ADHD関連のnote/ツイートをしている人）
- ターゲットの投稿内容
- 自分のアカウントの方向性

**処理内容**
1. ターゲットアカウントの投稿を分析
2. 共感・価値提供できる切り口を考える
3. 引用RTしてもらいやすいツイートを作成
   - 相手の投稿を褒める/共感する
   - 追加の価値を提供する
   - 相手がRTしたくなる内容

**出力**
```
output/x_posts/
  YYYYMMDD_engagement/
    target_analysis.md    # ターゲット分析
    quote_tweets.md       # 引用RT用ツイート案
```

**出力フォーマット例: quote_tweets.md**
```markdown
# 引用RT用ツイート案

## ターゲット: @adhd_life_tips

### 元ツイート
> ADHDあるある：締め切り3日前「まだ余裕」→前日「やばい」→当日「徹夜」

### 引用ツイート案

**案1（共感 + 解決策チラ見せ）**
これ、俺のことかと思った😇

最近はChatGPTに「今日やる最小タスク」を毎朝聞くようにしたら、前日徹夜がなくなった。

ADHDの「あとでやる」はAIで潰せる。

**案2（体験談）**
わかりすぎて泣いた。

でも最近発見したんだけど、ChatGPTに「このタスク、5分で何ができる？」って聞くと、なぜか着手できる。

脳のバグをAIで補うの、マジでおすすめ。
```

### 8. X記事宣伝エージェント

**目的**
自分のnote記事を宣伝するツイートを作成

**入力**
- 公開した/公開予定のnote記事
- 記事のポイント・想定読者
- 狙いドキュメント

**処理内容**
1. 記事の魅力を引き出す切り口を考案
2. 複数パターンのツイートを作成
3. スレッド形式のツイートも作成

**ツイートパターン**
- 問題提起型：「〇〇で悩んでませんか？」
- 体験談型：「これで人生変わった」
- 数字訴求型：「5つのプロンプト」
- Before/After型：「以前は〜、今は〜」

**出力**
```
output/x_posts/
  YYYYMMDD_promotion/
    article_tweets.md    # 記事宣伝ツイート案
```
