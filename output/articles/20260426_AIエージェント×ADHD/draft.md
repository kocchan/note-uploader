# 【ADHD向け】「やらなきゃ」をAIに丸投げ！実行機能が弱くても仕事が回るAIエージェント活用術

---

## 「頭ではわかってる。でも、動けない」

今日もまた、同じ1日を過ごしていませんか。

「週報、今日中に出さないと」
「あのメール、もう3日放置してる」
「明日の会議、何も準備してない」

全部わかってる。でも、着手できない。

気づいたら夜になって、「また今日もできなかった」と自己嫌悪。
その繰り返しに、正直もう疲れてきていませんか。

これ、意志が弱いわけじゃないんです。

ADHDの脳は「実行機能」と呼ばれる、タスクを始めて維持する力が弱い。
「やらなきゃ」という気持ちがあっても、脳がうまく着手モードに入れないことがある。

つまり、**頑張ろうとしても「脳の構造的な問題」で動けない**ことがあるわけです。

でも、ちょっと待ってください。

AIが「着手すること自体」を代わりにやってくれたら、どうなると思いますか？

---

## AIエージェントって何？——「聞く」から「やってくれる」へ

ChatGPTやClaudeを使ったことがある方なら、「AIに質問する」感覚はわかると思います。

でも、**AIエージェントはそれとはちょっと違います。**

わかりやすく例えると、こんな感じです。

- **従来のAI（チャット）** ＝ 料理レシピを教えてくれる人
- **AIエージェント** ＝ 材料を揃えて、料理まで作ってくれる家政婦さん

「何をすればいいか聞く」のではなく、「やっておいて」と任せる感覚です。

2026年、ChatGPTのAgent ModeやClaudeのProjects機能が進化して、個人でも「AIに仕事を任せる」ことが現実的になってきました。

ADHDの最大の悩み「着手できない」に対して、AIエージェントは**「着手する必要すらなくなる」** という解決策を提示してくれます。

## 【前提知識】ADHDがAIエージェントを使うべき、本当の理由

本題の前に、少しだけ寄り道させてください。

「プロンプトさえあれば使えるでしょ」と思うかもしれません。でも、なぜこれがADHDに効くのかを理解しておくと、活用の質が全然変わります。

ADHDの実行機能の弱さは、大きく3つに分かれます。

**①　着手困難**——「始める」のが苦手
**②　持続困難**——「続ける」のが苦手
**③　切り替え困難**——「止める/移る」のが苦手

AIエージェントのプロンプトが効くのは、主に①の着手困難に対してです。

通常、タスクを始めるには「何をするか考える → 優先順位をつける → 実際に手を動かす」という複数のステップが必要です。

ADHDの脳は、この**「考える」フェーズでエネルギーを使い果たしてしまう**ことがある。

だから「着手できない」んです。

でも、AIに「今日のタスクリストを作って」と頼んでおけば、考えるフェーズをAIが肩代わりしてくれます。あなたがやるのは「AIの出力を見て、最初の1タスクに手を付ける」だけ。

**着手のハードルが、劇的に下がります。**

実際に使ってみて驚いたのは、「AIが整理してくれると、なぜか動ける」ということです。自分で考えると動けないのに、AIに整理してもらうと動ける。不思議ですが、実行機能の観点からはちゃんと説明がつきます。

## 実践プロンプト①「朝の仕事リスト自動生成」

### どんな悩みを解決するか

「今日は何からやろう…」と朝30分考えて、気づいたら午前中が終わっている。

この悩み、ADHDあるあるじゃないでしょうか。

「今日やること」を自分で整理するのは、実行機能を使う典型的な作業です。朝一番にこれをやると、いちばんエネルギーがある時間帯を「考える」だけで消費してしまう。

このプロンプトは、前日の終業時に翌日の状況をAIに伝えておくと、翌朝「今日やること」リストを即座に提示してくれます。

### 使い方

1. **前日の終業前（5分でOK）** に、Claudeを開く
2. 以下のプロンプトをコピペして、`[]` の部分を自分の状況に書き換える
3. 翌朝、Claudeを開いてそのまま使う

### プロンプト

```
あなたは私のADHD対応の仕事サポートAIです。
私はADHDで、朝「何からやろう」と考えるのが特に苦手です。

今日（または明日）の仕事状況は以下の通りです：

【締め切りがあるもの】
[例：〇〇の資料を16時までに提出、〇〇さんへの返信]

【今日中にやりたいもの】
[例：先週から持ち越している□□、△△の確認]

【できればやりたいもの】
[例：◇◇の準備、メール整理]

これをもとに、以下を作ってください：
1. 今日やること（優先度順）を箇条書き5個以内
2. 「まず最初にやること」を1つだけ教えてください
3. 各タスクの目安時間を添えてください

余計な説明は不要です。リストだけ出してください。
```

### 実行例

**入力:**
```
【締め切りがあるもの】
・〇〇プロジェクトの進捗報告を17時までに上司にメール
・請求書の確認（今日中）

【今日中にやりたいもの】
・先週から放置している△△さんへの返信
・来週の会議の資料を少し進めたい

【できればやりたいもの】
・デスクの整理
・読みたかった社内資料を読む
```

**出力（イメージ）:**
```
【今日のタスクリスト】

① 請求書の確認（15分）← まずこれから
② 〇〇プロジェクト進捗報告メール作成（30分）
③ △△さんへの返信（10分）
④ 来週の会議資料 着手だけ（20分）
⑤ 上司へのメール送信確認（5分）

---
🟢 まず最初にやること：請求書の確認（15分）
```

### ポイント

- 「余計な説明は不要です」と入れることで、ADHDが読みやすいシンプルな出力になります
- 翌朝開いたときに迷わないよう、**「まずこれから」を1つだけ指示してもらう**のがコツ
- 「できればやりたいもの」をあえて入れることで、AIが適切に優先順位を下げてくれます

## 実践プロンプト②「メール文面の自動下書き」

### どんな悩みを解決するか

「メール1通書くのに、1時間かかった」

これ、笑えない実話です。謝罪メール、進捗報告、依頼メール……ADHDは「相手にどう思われるか」を考えすぎて、文章が進まなくなることがある。

しかも未読が溜まっていると、どのメールから返すかを考えるだけでパニックになる。

このプロンプトは、「言いたいことの箇条書き」を投げるだけで、ビジネスメールに変換してくれます。

### 使い方

1. 返信したいメールの「言いたいこと」をメモ書きで書き出す
2. 以下のプロンプトと一緒にClaudeに投げる
3. 出力をそのまま（または少し修正して）コピペして送信

### プロンプト

```
以下の内容をもとに、ビジネスメールの本文を作ってください。

【相手との関係】
[例：社内の上司 / 取引先の担当者 / 初対面の相手]

【メールの種類】
[例：進捗報告 / 謝罪 / 依頼 / 返信]

【言いたいこと（箇条書きでOK）】
・[要点1]
・[要点2]
・[要点3]

【トーン】
[例：丁寧に / 簡潔に / 少し謝罪を含めて]

条件：
- 件名も考えてください
- 署名は含めなくていいです
- 250文字前後でお願いします
```

### 実行例

**入力:**
```
【相手との関係】社内の上司
【メールの種類】進捗報告（遅れの報告）
【言いたいこと】
・〇〇の資料、今日中に出すと言ったけど間に合わなかった
・明日の午前中には必ず出す
・ご迷惑をおかけして申し訳ない
【トーン】丁寧に謝罪を含めて
```

**出力（イメージ）:**
```
件名：【〇〇資料】提出遅延のご連絡

〇〇部長

お世話になっております。

本日中にご提出をお約束しておりました〇〇の資料について、
大変申し訳ございませんが、本日中の提出が難しい状況です。

明日の午前中には必ずご提出させていただきます。
ご迷惑をおかけし、誠に申し訳ございません。

引き続きよろしくお願いいたします。
```

### ポイント

- 「言いたいこと」は箇条書き3行でOK。文章にしなくて大丈夫。
- 謝罪メールこそこのプロンプトが真価を発揮します。感情的になりがちな謝罪を、適切なトーンに整えてくれます。
- 出力の**1〜2文を自分の言葉に直す**と、より自然になります。全部AIの文章のまま送ることへの抵抗がある人はここだけ手を入れてみてください。

## 実践プロンプト③「会議前5分の自動準備」

### どんな悩みを解決するか

会議の10分前に「あ、今日会議あった」と気づく。

準備ゼロで出席して、「あれ、前回何を決めたんだっけ」とオロオロする。会議が終わったあと「あれ確認しておけばよかった」と後悔する。

ADHD的に、会議は地雷原みたいなものです。

このプロンプトは、会議名と参加者をAIに投げるだけで、「何を確認すべきか」「どんな準備が必要か」を5分でまとめてくれます。

### 使い方

1. カレンダーで会議の通知が来たら（5分前でもOK）Claudeを開く
2. 会議名・参加者・わかっている内容を入力
3. 出力を見ながら、必要な資料だけ開いておく

### プロンプト

```
私はADHDで会議の準備が苦手です。
以下の会議に参加する前に、何を準備・確認すべきかを教えてください。

【会議名・目的】
[例：週次定例 / 〇〇プロジェクト進捗確認]

【参加者】
[例：上司、同僚の△△さん]

【わかっていること・前回の流れ】
[例：先週〇〇を宿題にもらった / 今回は□□を報告する予定]

出力してほしいもの：
1. 会議前に確認すべきこと（3個以内）
2. 自分が話す可能性があること（2個以内）
3. 会議中にメモしておくべき項目
```

### 実行例

**出力（イメージ）:**
```
【会議前に確認すること】
① 先週もらった「〇〇の件」の宿題ステータスを確認
② □□プロジェクトの最新数字をメモしておく
③ 前回の議事録の最後の「決定事項」欄を見直す

【自分が話す可能性があること】
・〇〇の現状報告
・来週のスケジュール確認

【会議中にメモしておく項目】
・次回の宿題として振られたこと
・決定事項（誰が・何を・いつまでに）
```

### ポイント

- 「わかっていること」は少なくても大丈夫。「なし」でも意外と使える出力が返ってきます。
- **会議が始まる直前、トイレに立ったタイミング**でスマホで使うのがおすすめです。会議の前に「何を喋ればいいか」がクリアになるだけで、全然違います。

## 実践プロンプト④「タスク緊急度×重要度の自動仕分け」

### どんな悩みを解決するか

タスクが多い日は、何から手をつければいいかわからなくてパニックになる。

そして「全部同じくらい大事な気がする」「全部今日中な気がする」という謎の感覚に陥り、結局何もできずに終わる。

このプロンプトは、頭の中のタスクをそのまま投げると、緊急度と重要度で自動仕分けしてくれます。

### プロンプト

```
以下のタスクを「緊急度×重要度マトリックス」で仕分けしてください。

【今日のタスク（思いついたままでOK）】
[タスクを箇条書きで全部書く]

仕分けの基準：
- 第1象限（緊急・重要）：今日中に必ずやる
- 第2象限（緊急でない・重要）：今日やれればやる
- 第3象限（緊急・重要でない）：誰かに任せるか後回し
- 第4象限（緊急でない・重要でない）：今日はやらない

出力形式：
各象限に振り分けたタスク一覧と、「今日最初にやること1つ」を教えてください。
```

### 実行例

**入力（思いついたまま）:**
```
・週報（金曜締め切り、今日は水曜）
・Aさんへのメール返信（3日前から放置）
・来月の予算申請（再来週締め切り）
・デスクの整理
・上司との面談の準備（明日）
・読み途中の業界レポート
```

**出力（イメージ）:**
```
【第1象限】緊急・重要：今日中に必ずやる
→ Aさんへのメール返信
→ 上司との面談の準備（明日なので今日中）

【第2象限】緊急でない・重要：今日やれればやる
→ 週報（金曜締め切り・今日着手推奨）

【第3象限】後回しOK
→ 来月の予算申請

【第4象限】今日はやらない
→ デスクの整理
→ 業界レポート

---
🟢 今日最初にやること：Aさんへのメール返信（10分）
```

### ポイント

- 「全部大事な気がする」ときこそ使ってください。AIは感情なしに優先順位をつけてくれます。
- 出力に納得いかない部分は「〇〇は実は今日中です」と追加で伝えると調整してくれます。

## 実践プロンプト⑤「週次レポートの自動生成」

### どんな悩みを解決するか

「今週何やったっけ…」と毎週金曜に頭を抱える。

ADHDは短期記憶が弱い。1週間前に自分がやったことを思い出すのは、ただでさえ大変なのに、それを上司向けに整えた文章にするのは二重苦です。

このプロンプトは、今週やったことをメモ書きで投げるだけで、上司向けの週次レポートを自動生成してくれます。

### 使い方

コツは**その日にやったことをメモしておくこと**。Slackのメッセージ、カレンダーの予定、送受信メールを見返しながら、箇条書きで記録しておくだけでOKです。

### プロンプト

```
以下の今週の作業内容をもとに、上司向けの週次レポートを作成してください。

【今週やったこと（箇条書きでOK）】
[内容を書く]

【進行中のプロジェクト】
[例：〇〇プロジェクト（△フェーズ）]

【来週やる予定のこと】
[例：□□の着手、◇◇の完了]

【困っていること・相談したいこと（あれば）】
[例：なし / △△について相談したい]

出力条件：
- 300〜400文字
- 「今週の成果」「進行中」「来週の予定」の3段構成
- 丁寧なビジネス語で
```

### ポイント

- 「今週やったこと」は汚くて大丈夫。「〇〇の件、Aさんとやりとり」程度でも機能します。
- **週の途中（水曜・木曜）からメモを始めても十分間に合います。**
- 出力を見て「こんなことやってたんだ」と気づくこともあります。自己肯定感の回復にも意外と効く。

## 応用編：Claude Projectsで「自分専用AIエージェント」を育てる

5つのプロンプトを使ってみたら、次のステップとして**Claude Projects**を試してみてください。

Claude Projectsとは、「自分の仕事情報を事前に覚えてくれるClaudeの専用スペース」です。

普通のClaudeは、毎回「私はADHDで…」「仕事は〇〇で…」と説明し直す必要があります。でもProjectsに情報を登録しておくと、**毎回説明しなくてもよくなります。**

### 登録しておくと便利な情報

```
【自己紹介（AIへの説明）】
私はADHDがあり、着手することと優先順位をつけることが特に苦手です。
シンプルな出力を好みます。説明は短く、リストを使ってください。

【仕事の基本情報】
職種：〇〇
よく関わる相手：上司の〇〇さん（厳しめ）、同僚の△△さん
よく書くメールの種類：進捗報告、社内依頼

【フォーマット指定】
・箇条書き優先
・長い文章より短い文章で
・「まず何をするか」を必ず1つ教えて
```

これを登録しておくだけで、プロンプトがより短くて済むようになります。
