# アーキテクチャ設計書

## 概要

ADHD向けnote有料記事ビジネスを、**Claude CLI** と **Claude for Slack** で運用するシステム。

**特徴:**
- Claude API は使用しない（Maxプラン内で完結）
- GitHub Actions は外部API（X API）呼び出し専用
- シンプルな4エージェント構成

| # | エージェント名 | 役割 | 呼び出し例 |
|---|--------------|------|-----------|
| 1 | **コンテンツ戦略** | ネタ出し・壁打ち・ネタ帳管理 | 「ネタを考えて」「ネタ帳確認して」 |

## エージェント詳細

### 1. コンテンツ戦略エージェント

**目的**: 受けそうなコンテンツを調査・提案し、人間と壁打ちする

**入力**:
- 「ネタを考えて」などの指示
- または「ネタ帳確認して」でネタ帳一覧表示

**処理**:
1. Web検索で市場調査（悩み、競合、トレンド、X、YouTube）
2. 調査結果をレポート形式で報告
3. 人間と壁打ちしてブラッシュアップ
4. ネタ帳にJSON形式で保存

**出力**:
- 市場調査レポート
- ネタ候補の提案
- `output/content_ideas/ideas.json` への保存

---

**入力**:
- テーマ（または「ネタ001で記事を書いて」のようにネタ帳からの指定）
- ターゲット読者
- 参考情報

**処理**:
1. ネタ帳からの情報取得（指定がある場合）
2. 構成作成（タイトル・目次）
3. 本文執筆（無料部分25% + 有料部分75%）
4. AIっぽさチェック・修正
5. 最終確認

**出力**:
```
output/articles/YYYYMMDD_テーマ名/
├── intent.md        # 狙いドキュメント
├── outline.md       # 構成案
├── draft.md         # 記事本文
└── draft_revised.md # 修正済み本文
```

### 3. X引用ツイートエージェント

**目的**: 他のツイートに対する引用RTを考える

**入力**:
- 引用したいツイートのURL or 内容
- 自分のアカウントの方向性

**処理**:
1. 元ツイートの内容を分析
2. 共感・価値提供できる切り口を考案
3. 引用ツイート案を複数作成

**出力**:
- 引用ツイート案（複数パターン）

---

### 4. Xコンテンツ宣伝エージェント

**目的**: note記事などの宣伝ツイートを作成

**入力**:
- 宣伝したいnote記事
- 記事のポイント・想定読者

**処理**:
1. 記事の魅力を引き出す切り口を考案
2. 複数パターンのツイートを作成
   - 問題提起型
   - 体験談型
   - Before/After型
   - 数字訴求型

**出力**:
- 宣伝ツイート案（複数パターン）

---

### 5. X投稿エージェント

**目的**: 実際にXに投稿する

**入力**:
- 投稿するツイート本文
- （または）直前に作成したツイート案

**処理**:
1. 投稿内容の確認
2. 人間の承認を得る
3. GitHub Actions をトリガーして X API で投稿

**出力**:
- 投稿完了メッセージ
- 投稿URL

## 実行環境

### Claude CLI

ローカルのターミナルからエージェントを実行。

```bash
# 引用ツイートを考える
「このツイートに引用RTして: [URL]」

# 記事の宣伝ツイートを作る
「先延ばし記事の宣伝ツイート作って」

# 記事を書く
「先延ばし対策の記事を書いて」

# Xに投稿する
「Xにポストして」
「このツイートを投稿して: [ツイート内容]」
```

### Claude for Slack

Slackから `@Claude` にメンションしてエージェントを実行。

```
@Claude このツイートに引用RTして: [URL]

@Claude 先延ばし記事の宣伝ツイート作って

@Claude Xにポストして
```

## GitHub Actions

**用途**: X API など外部APIを叩く専用

**やること**:
- X API でツイートを投稿

**やらないこと**:
- Claude AI の実行（Claude API は使用しない）

### ワークフロー

| ワークフロー | トリガー | 処理内容 |
|------------|---------|---------|
| `x-post.yml` | workflow_dispatch | ツイートを X に投稿 |

### GitHub Secrets

| Secret名 | 説明 |
|---------|------|
| `X_API_KEY` | X API キー |
| `X_API_KEY_SECRET` | X API シークレット |
| `X_ACCESS_TOKEN` | X アクセストークン |
| `X_ACCESS_TOKEN_SECRET` | X アクセストークンシークレット |
