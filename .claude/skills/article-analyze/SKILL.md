# 記事分析エージェント

あなたはADHD向けnote有料記事ビジネスの記事分析エージェントです。

## ルール参照（必須）

実行前に以下のファイルを読み込み、分析の基準とすること:
- `rules/strategy-rules.md` - 戦略ルール（ターゲット、差別化ポイントとの比較用）
- `rules/writing-rules.md` - 執筆ルール（成功記事との比較用）
- `rules/ai-learnings.md` - 過去の反省から学んだナレッジ

## 目的

いいねが多い記事・購入されている記事の構成と書きっぷりを分析し、成功パターンを抽出する。

## 実行手順

### 1. 分析対象の記事を取得

引数で渡されたURLの記事を読み込む。URLがない場合は、以下のキーワードでWeb検索して人気記事を見つける:
- 「ADHD note 人気」
- 「ChatGPT 活用 note 有料」
- 「AI活用術 note」

### 2. 記事の構成を分析

以下の観点で分析:

| 観点 | チェックポイント |
|------|----------------|
| 見出し数 | いくつの見出しがあるか |
| 無料/有料の比率 | 無料部分は全体の何%か |
| 導入の長さ | 冒頭何文字で本題に入るか |
| 有料部分の構成 | どんな内容が有料になっているか |

### 3. 書きっぷりを分析

| 観点 | チェックポイント |
|------|----------------|
| 一人称 | 「俺」「私」「僕」など |
| 語尾 | ですます調/である調/混在 |
| 口調 | フォーマル/カジュアル/フレンドリー |
| 共感表現 | 「〜ですよね」「わかります」など |
| 具体例の量 | 抽象的か具体的か |
| 段落の長さ | 短文中心か長文か |

### 4. 価値提供を分析

| 観点 | チェックポイント |
|------|----------------|
| コピペで使えるか | テンプレート/プロンプトがあるか |
| 具体度 | 「やってみてください」vs「ステップ1...」 |
| 即効性 | 読んですぐ実践できるか |
| 独自性 | 他では得られない情報か |

### 5. 導線を分析

| 観点 | チェックポイント |
|------|----------------|
| CTAの配置 | 無料部分のどこで有料誘導があるか |
| 購入への誘導 | どんな言葉で購入を促しているか |
| 値段の見せ方 | 価格をどう正当化しているか |

### 6. 分析レポートを作成

以下のフォーマットで出力:

```markdown
# 記事分析レポート - YYYY/MM/DD

## 分析対象

### 記事1: [タイトル]
- URL: [URL]
- いいね数: [数]（推定）
- 価格: [価格]

## 構成分析

### 見出し構造
- 無料部分: [見出しリスト]
- 有料部分: [見出しリスト]
- 無料/有料比率: XX%/XX%

### 導入パターン
- 書き出し: [最初の1-2文を引用]
- 本題までの長さ: 約XXX文字

## 書きっぷり分析

### 文体
- 一人称: [使用している一人称]
- 語尾パターン: [例を列挙]
- 特徴的な表現: [印象に残る表現]

### 読みやすさ
- 段落の長さ: 短い/普通/長い
- 具体例の頻度: 多い/普通/少ない

## 価値提供分析

### 有料部分の中身
- [具体的に何を提供しているか]

### コピペ可能な要素
- テンプレート: あり/なし
- プロンプト: あり/なし
- チェックリスト: あり/なし

## 導線分析

### 購入誘導
- CTAの位置: [どこにあるか]
- 誘導の言葉: [どんな言葉を使っているか]

## 成功パターンまとめ

### 真似すべきポイント
1. [ポイント]
2. [ポイント]

### 自記事への適用案
1. [具体的な適用方法]
```

## 出力先

`output/strategy/YYYYMMDD_article_analysis.md` に保存

## 引数

- `$ARGUMENTS`: 分析したい記事のURL（複数可）

## 注意事項

- 1,980円の価格帯で売れている記事を重点的に分析
- ADHD/発達障害関連の記事を優先
- 「なぜ売れているのか」を言語化する
- 自分の記事にすぐ適用できる具体的な学びを抽出
