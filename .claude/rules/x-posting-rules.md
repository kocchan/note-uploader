## 成功パターン

### 1. タイトル型投稿

**フォーマット:**
```
[キャッチーなタイトル]

「[読者の悩み・疑問]」

[共感を誘う一言]

[問題点を具体的に列挙]

[解決策の予告]
...
```

**例:**
```
Claude Codeで作ったデザイン、「ダサい」と思ったことありませんか？MCPで全部解決する完全ガイド

「Claude Codeでデザインしたんだけど……絶妙にダサい。」
...
```

### 2. 問題提起 → 解決型

**フォーマット:**
```
[読者の悩み・やりたいこと]で、[現実の壁]

[具体的な数字で現実を見せる]
・選択肢A → [コスト/デメリット]
・選択肢B → [コスト/デメリット]
・選択肢C → [コスト/デメリット]

[でも解決策がある、という予告]
...
```

**例:**
```
「うちにもAIチャットボット欲しいんだけど……」で、見積もり取ったことありますか？

チャットボットSaaS → 月額30万円
開発会社に外注 → 初期費用100万〜
自社開発 → いや、エンジニアおらんし
...
```

## 投稿テクニック

### DO（やるべきこと）

1. **タイトルを先に出す**
   - 記事のタイトルをそのままツイートの冒頭に
   - 「〜で全部解決する完全ガイド」のような訴求力のあるタイトル

2. **具体的な数字を入れる**
   - 金額: 「月額30万円」「初期費用100万〜」
   - 回数: 「コピペ3回」「7つのプロンプト」
   - 期間: 「その日のうちに」「1ヶ月で」

3. **読者への問いかけ**
   - 「〜と思ったことありませんか？」
   - 「〜で、見積もり取ったことありますか？」

4. **「...」で続きを匂わせる**
   - ツイートの最後を「...」で終わらせ、記事への誘導

5. **Quote RTで自分の投稿を広める**
   - 元投稿に一言コメントを添えてQuote RT
   - 「〜に興味ある方々は、マジで必見です。」

6. **口語的な表現を混ぜる**
   - 「マジで必見」「いや、〜おらんし」
   - 堅すぎず、カジュアルすぎず

### DON'T（避けること）

- 「買ってください」と直接言う
- URLだけの投稿
- 長すぎる説明
- 絵文字の使いすぎ
- ハッシュタグの乱用（2-3個まで）
- **上から目線・押し付けがましい表現**
  - NG例: 「心当たり、ありますよね。」「わかりますよね？」
  - 読者を見透かしたような言い方は鼻につく

## ツイート文体（重要）

ツイートは「記事」ではなく「つぶやき」。自然な口語体で書く。

### 基本ルール

1. **「!」を使う**
   - 「〜になった!」「〜だった!」
   - 堅い「。」で終わらせない

2. **「...」「...!」を使う**
   - 「まさにこれ...!」「わかりすぎる...」
   - 感情の余韻を残す

3. **絵文字を適度に使う**
   - 😇 自虐・苦笑い
   - 👍 おすすめ・良かった
   - 😭 共感・救われた
   - 使いすぎはNG（1-2個まで）

4. **口語表現を使う**
   - 「マジで」「だいぶ」「全然違う」
   - 「〜なった!」「〜だった!」（過去形で体験談風に）

5. **短く切る**
   - 一文を長くしない
   - 改行を多めに

### NG文体

```
❌ 〜するようになりました。（堅すぎる）
❌ 〜のが効く。（教訓っぽい）
❌ 〜だと思います。（弱い）
❌ 〜ものです。（AIっぽい）
```

### OK文体

```
✅ 〜になった!
✅ 〜だった...!
✅ マジで変わった👍
✅ これ、僕のことだ...😇
```

### 例

```
❌ Before（堅い）
ChatGPTに聞くようにしたら、完璧じゃなくても動けるようになりました。
「完璧」の前に「とりあえず」を挟むのが効く。

✅ After（ツイートらしい）
ChatGPTに聞いてから始めるようにしたら、完璧じゃなくても動けるようになった!
```

## X API自動投稿

### 概要

`/x-post` スキルでX APIを使った自動投稿が可能。

### 料金

- 従量課金: **$0.01/投稿**
- 事前にX Developer Portalでクレジット購入が必要

### ワークフロー

```
1. /x-promotion or /x-engagement でツイート案作成
2. 人間がレビュー・承認
3. /x-post "承認されたツイート" で投稿
4. 投稿履歴は output/x_posts/post_history.md に自動記録
```

### コマンド

```bash
# 直接投稿
/x-post "ツイート本文"

# 引用RT
/x-post "ツイート本文" --quote TWEET_ID

# リプライ
/x-post "ツイート本文" --reply-to TWEET_ID
```

### 注意事項

- **必ず人間の承認を得てから投稿する**
- **Twitter換算文字数で280文字制限**（下記参照）
- 投稿前にツイート文体ルールを確認
- APIエラー時は15分待ってリトライ

### 文字数カウントルール（重要）

Twitterの文字数カウントは単純な文字数ではない:

| 文字種 | カウント |
|--------|----------|
| 半角英数 | 1文字 |
| 日本語 | **2文字** |
| 絵文字 | **2文字** |
| URL | **23文字固定** |

**目安:**
- 日本語ツイート: 単純文字数 **140文字以内**
- URL含む場合: 単純文字数 **120文字以内**

**超過すると403エラーで投稿失敗する**
