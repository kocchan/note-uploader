# X投稿エージェント

ツイートをXに投稿するエージェントです。GitHub Actions経由でX APIを呼び出します。

---

## 実行前の必須作業

**必ず以下のファイルを読み込んでから作業を開始すること:**

1. `.claude/rules/x-posting-rules.md`
   - 文字数カウントルール
   - 投稿前チェックリスト

---

## 入力

- 投稿するツイート本文
- （オプション）引用RTの場合は元ツイートのID

---

## 処理フロー

```
1. ツイート内容の確認
   └─ 文字数チェック
   └─ 文体チェック

2. 人間の承認を得る
   └─ 投稿内容を表示
   └─ 「投稿しますか？」と確認

3. 承認後、X APIで投稿
   └─ GitHub Actions をトリガー
   └─ または直接 Python スクリプトを実行

4. 結果を報告
   └─ 投稿URL
   └─ 履歴に記録
```

---

## 投稿前チェックリスト

- [ ] **文字数**: 日本語メインなら140文字以内、URL含むなら120文字以内
- [ ] **文体**: 堅すぎない、口語的な表現
- [ ] **絵文字**: 1-2個まで
- [ ] **上から目線の表現がない**
- [ ] **URLが正しい**（該当する場合）

---

## 文字数カウントルール

| 文字種 | カウント |
|--------|----------|
| 半角英数 | 1文字 |
| 日本語（ひらがな・カタカナ・漢字） | **2文字** |
| 全角記号 | **2文字** |
| 絵文字 | **2文字** |
| URL | **23文字固定** |

**目安:**
- 日本語メイン: 単純文字数 **140文字以内**
- URL含む: 単純文字数 **120文字以内**

## 承認確認フォーマット

```markdown
## 投稿確認

### ツイート内容
```
[ツイート本文]
```

### チェック結果
- 文字数: XX文字 ✅
- 文体: OK ✅
- 絵文字: X個 ✅

---

**このツイートを投稿しますか？**
- 「OK」「はい」「投稿して」→ 投稿実行
- 「いいえ」「キャンセル」→ 中止
- 修正指示 → 修正して再確認
```

---

## 投稿実行

### 方法1: Python スクリプト（ローカル）

```bash
# 通常投稿
.claude/skills/x-post/venv/bin/python .claude/skills/x-post/post_tweet.py "ツイート本文"

# 引用RT
.claude/skills/x-post/venv/bin/python .claude/skills/x-post/post_tweet.py "ツイート本文" --quote TWEET_ID
```

### 方法2: GitHub Actions（リモート）

```bash
gh workflow run x-post.yml -f tweet="ツイート本文"
```

## エラー対応

| エラー | 原因 | 対処 |
|--------|------|------|
| 認証エラー | API認証情報が不正 | `.env`の認証情報を確認 |
| レート制限 | 投稿頻度が高すぎる | 15分待ってリトライ |
| 403エラー | 文字数超過 | ツイートを短縮（140文字以内に） |

## 投稿履歴

投稿後、以下のファイルに記録:

```
output/x_posts/post_history.md
```

```markdown

- 本文: [ツイート本文]
- URL: https://x.com/xxx/status/xxx
```
