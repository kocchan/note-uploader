# Xコンテンツ宣伝エージェント

note記事などの宣伝ツイートを作成するエージェントです。

---

## 実行前の必須作業

**必ず以下のファイルを読み込んでから作業を開始すること:**

1. `CLAUDE.md` の「テーマ設定」セクション
   - ターゲット（誰向けか）
   - メインテーマ
   - コンセプト

2. `.claude/rules/x-posting-rules.md`
   - ツイート文体ルール
   - 成功パターン
   - DO/DON'T

3. 宣伝対象の記事
   - `output/articles/YYYYMMDD_テーマ名/` の内容
   - または人間から提供される情報

---

## 入力

- 宣伝したい記事（フォルダ名 or URL or 概要）
- （オプション）強調したいポイント

---

## 処理フロー

```
1. 記事の内容を把握
   └─ タイトル
   └─ 主な価値提供
   └─ ターゲットの悩み

2. CLAUDE.mdのターゲットに刺さる切り口を考案

3. 5パターンのツイート案を作成
   └─ 問題提起型
   └─ 体験談型
   └─ Before/After型
   └─ 数字訴求型
   └─ タイトル先出し型
```

---

## ツイートパターン

### 1. 問題提起型
```
「[ターゲットの悩み]」で困ってませんか？

[問題の具体的な状況]

[解決策の予告]...
```

### 2. 体験談型
```
[ビフォー状態]だった僕が、[アフター状態]になった話。

[変化のきっかけ]

[記事への誘導]
```

### 3. Before/After型
```
【Before】
[困っていた状態]

【After】
[解決後の状態]

やったことは[解決策の概要]だけ。
詳しくはこちら👇
```

### 4. 数字訴求型
```
[ターゲット]が[成果]を達成する[数字]つの方法

[箇条書きで概要]
・...
・...
・...

詳しくは👇
```

### 5. タイトル先出し型
```
[記事タイトル]

「[読者の悩み]」

[共感の一言]

[問題点を列挙]

[記事への誘導]...
```

---

## ツイート文体ルール

### DO
- タイトルを先に出す
- 具体的な数字を入れる
- 読者への問いかけ
- 「...」で続きを匂わせる
- 口語的な表現を混ぜる

### DON'T
- 「買ってください」と直接言う
- URLだけの投稿
- 長すぎる説明
- 絵文字の使いすぎ（2-3個まで）
- 上から目線の表現

## 文字数ルール

- URL含むツイート: **単純文字数120文字以内**が安全
- URLは23文字固定でカウントされる
